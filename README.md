Практические занятия 7–11: аутентификация, JWT, React и RBAC

В рамках практических занятий с 7 по 11 разработана система аутентификации и авторизации с управлением товарами и пользователями. Проект включает бэкенд на Node.js с Express и фронтенд на React (Vite, TypeScript).

Седьмая практика. Реализована базовая аутентификация с хешированием паролей через bcrypt. Есть маршруты регистрации и входа, полный CRUD для товаров. Пароли хранятся в виде хеша, при входе выполняется сравнение. Для товаров можно загружать изображения.

Восьмая практика. Добавлены JWT access-токены. После входа сервер отдаёт токен, клиент сохраняет его и отправляет в заголовке Authorization. Есть middleware проверки токена и маршрут /api/auth/me для текущего пользователя.

Девятая практика. Добавлены refresh-токены: access живёт 15 минут, refresh — 7 дней. Когда access истекает, клиент вызывает /api/auth/refresh и получает новую пару; старый refresh удаляется, новый сохраняется (ротация). На фронтенде перехватчик Axios при ответе 401 обновляет токены и повторяет запрос.

Десятая практика. Фронтенд на React: регистрация, вход, создание и редактирование товаров, список с удалением. Токен подставляется в запросы автоматически; продление access по refresh для пользователя незаметно.

Одиннадцатая практика. Роли: user — только просмотр товаров; seller — создание и редактирование товаров; admin — всё у продавца плюс управление пользователями (роли, блокировка, удаление любых товаров). Есть middleware по ролям и маршруты для админов. В интерфейсе у администратора есть раздел «Пользователи».

При старте сервера, если такого пользователя ещё нет, создаётся администратор по умолчанию: email admin@example.com, пароль admin123 (можно переопределить переменными DEFAULT_ADMIN_EMAIL и DEFAULT_ADMIN_PASSWORD). Новые пользователи при регистрации получают роль user; роли меняет администратор.

Структура: папка backend — сервер, файл server.js. Папка frontend — клиент.

Запуск. Нужны Node.js и npm или pnpm. Зависимости ставьте в корне, в backend и во frontend.

Один терминал из корня репозитория: npm run dev — поднимаются бэкенд и фронтенд вместе.

Два терминала. В первом: cd backend, npm install, npm run dev — сервер на http://localhost:3000, Swagger на http://localhost:3000/api-docs. Во втором: cd frontend, npm install, npm run dev — приложение в браузере, обычно http://localhost:5174, запросы к API идут на бэкенд через прокси.
